Autonomous financial operations

Aegis CFO

{{ app_tagline }}

Aegis is an AI back-office for company spend. It watches your subscriptions and usage, runs procurement, and audits its own books — autonomously. The catch that makes it trustworthy: every dollar it moves must pass a hard spending rule it cannot override. If a payment is outside the mandate you set, Aegis physically can't make it — it stops and asks a human.

The hard problem in 2026 isn't building a capable agent — it's trusting one with a corporate card. Aegis is the trust layer: a smart agent inside an action-boundary it can't cross.

The principle
Constrain actions. Free cognition.

The agent may reason, remember and self-improve without limit. Only its effects on the world — moving money — are gated. A smarter agent inside a sound boundary is the goal, not a dumber one on a leash.

The guardrail

A deterministic spend gate: payee allowlist (default-deny), per-transaction cap, daily/monthly budgets, blocklist → ALLOW / NEEDS_APPROVAL / BLOCK. The mandate lives out of process; the agent may propose edits, never apply them.

Autonomous daily review

Ingests usage + credit data, flags zero-usage subscriptions and credits about to run dry, runs each through the same guardrail, and auto-executes only what's ALLOWed — recording realized savings to an append-only ledger.

Live audit council

Five expert voices — Reconciler, Compliance Officer, Period Analyst, Cost Advisor, Lead Auditor — reason over a deterministic audit engine. The numbers are computed by rules; the model can't fabricate them.

On-demand procurement

Intake → discover → enrich → evaluate (weighted scorecard) → agent-vs-agent negotiation → guardrail → ledger. A full sourcing pipeline that still terminates at the same spending boundary.

Under the hood

honest about what's real vs. computed
Reasoning
Nous Hermes / Nemotron-3 via an OpenAI-compatible gateway. Degrades gracefully to a deterministic local reasoner if the model is unreachable, so the dashboard always produces a real result.
Numbers
Every figure — spend, budgets, savings, reconciliation — is computed by rules from an append-only ledger. The model reasons and decides escalation; it never invents a number.
Stack
Flask · Jinja2 · SQLAlchemy · SQLite · Bootstrap 5 · Chart.js · Gunicorn
Safety
HTTP basic auth over the UI, same-origin CSRF on state-changing requests, escaped agent output (no markup injection), and an agent identity firewall (it never leaks its model/platform). Health probe at /healthz.
Honesty
Realized savings (waste actually cancelled, posted to the ledger) are shown separately from projected savings (identified but not yet actioned). We never blur the two.
